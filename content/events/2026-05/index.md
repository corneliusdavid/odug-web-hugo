This month will not focus so much on what's new in Delphi programming as it will in what not to do when managing multiple Windows servers in the cloud. Maintaining health of both the physical bodily kind and a one-man application cloud is a tricky balancing act!

<!--more-->

---

Due to an over-confident programmer, bolstered by ever-positive AI responses, an attempt to migrate an old, single Remote Desktop server into three newer ones with load-balancing, proved to be a two-week long saga of late nights, troubleshooting, and running endless PowerShell scripts. Programming was all but set aside in order to keep ~60 users connected while various obscure Windows server issues were researched at night. 

Discussion will briefly cover Windows licensing, integrating old servers with new, some useful PowerShell scripts learned, and the joys of managing user profile disks. (I stress "briefly" because this is a Delphi programming group, not a Windows Server Management or IT group!)

We'll also review why a file-upload solution presented at the February meeting didn't stick and a couple of possible new ideas, one of which involves a Delphi program still in the making.

Come listen and commiserate--and enjoy good Italian food!
